About

Matching question is one of the core question types of the Axonify platform. I was tasked to design a mobile only experience for this question type. The desktop “responsive” experience was not working with our customers and it was very confusing to use.

{image}

Problem

With the app using the same code-base for both desktop and mobile applications, the user experience for one of the more complicated question types, “Matching Question” was suffering. I was tasked with creating a mobile only view for this question type.

Constraints

Cannot vary functionality wise. Still need the ability to zoom images, show your current matches and see correct matches if user answers incorrectly.

{image}

High-Level Goals

  • Mobile only experience
  • Do not vary far from current implementation

Research

  • Examine some online matching games for examples of how some actual games treat matching
  • Reading the customer feedback “I can’t figure this out”, “This UI sucks” :)
{image}

Solution

I decided on a 2-up view with the ability to cycle through different answers. The idea is to cycle the items until you find the correct answers. Hitting the giant match button will place the matches into a preview area where you can review or remove matches. Once you have matched all, you continue with confidence based answer.
