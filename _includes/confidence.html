Confidence Based Question Answering

Concept Creation • Prototyping • Visual Design • Front End Development

Used buy ~500K users everyday

{image}

About

Microlearning questions is one of the main functions in the Axonify platform. Short 3-5 minute training bursts allow users to be trained quickly over a period of time.

{image}

Problem

To better track user topic knowledge, there was a need to add confidence based answering to the microlearning experience. By asking how confident the user is on the answer, the system could better track knowledge retention vs knowledge decay . As one of the core UX flows of the platform the solution could not add any more time to the end user experience.

{image}

High-Level Goals

  • Add “High, Medium, Low” confidence based answer option to all question types
  • Do not add, or try not to add, any additional time to the microlearning experience
  • Support both desktop and mobile experiences

Research

  • Research industry standards around confidence question answering
  • Some competitive analysis
  • A/B testing with internal and external users on different UX flows.
  • Define easy language to assess confidence. This had to work with a broad target audience (teens to adults)
{image}

Solution

After some rounds of testing, showing confidence choices when you select your answer was by far the clear winner. Showing confidence on question answering did not add any extra time or clicks to the UX. Only by selecting your confidence level could you advance to the next question. If answered incorrectly, you would see the correct answer and then be able to move on.

Key Features

  • Simple UI
  • Clear easy-to-understand choices
  • Clean interaction when selecting answers
