Confidence Based Question Answering

Concept Creation • Prototyping • Visual Design • HTML/CSS

About

Microlearning questions is one of the main driver within the Axonify platform. Short 3-5 minute training bursts per day allow users to be trained quickly over a short period of time. This confidence based answering system is how we do it.

Problem

To better track user topic knowledge, I was asked to add confidence based answering to the microlearning experience. By asking how confident the user is on the answer, the system could better track knowledge retention vs knowledge decay based on brain science research. As one of the core UX flows of the platform the solution, a main goal was that I could not add any more time to the end user experience. 3 mins a day is all it should take.

Early Flow - Mobile

High-Level Goals

  • Add confidence based answer option to all question types
  • Do not add any additional time to the microlearning experience
  • Support both desktop and mobile experiences

Research

  • Some competitive analysis, not much, we are a disruptor in the field.
  • A/B testing with internal users and external customers on different UX flows and verbiage.
  • Define easy language to assess confidence. This had to work with a broad target audience (teens to adults)
Different Verbiage & experience ideas - Mobile

Solution

After some rounds of testing, showing confidence choices when you select your answer was by far the clear winner. Showing confidence on question answering did not add any extra time or clicks to the UX. Only by selecting your confidence level could you advance to the next question. If answered incorrectly, you would see the correct answer and then be able to move on.

Key Experiences

  • Simple UI
  • Clear easy-to-understand choices
  • Clean interaction when selecting answers
  • Desktop & Mobile friendly
